📦 Overview

This software serves as a loader for the Firefly Aggregate Downloader. It helps users manage their software by checking for updates and providing access to the latest versions. The tool ensures your files stay current without manual intervention. By using this hub, you avoid broken links and outdated files. The interface displays the release status, allowing you to trigger a download with one click.

🛠️ Requirements

  • Operating System: Windows 10 or Windows 11
  • Memory: 4GB RAM minimum
  • Storage: 200MB available disk space
  • Internet: Stable connection for update checks

📥 Setup and Installation

Follow these steps to get the software up and running on your computer.

  1. Visit the release page: Go to the official releases page to view all available versions.
  2. Select a version: Pick the build labeled as Latest.
  3. Download the file: Click on the link ending in .exe to save it to your computer.
  4. Run the application: Locate the file in your downloads folder and double-click it.
  5. Grant permissions: If Windows shows a security prompt, click Run or Yes to launch the loader.

⚙️ How to use

Once the hub finishes loading, it will run a check of the server. If a new version exists, the main button will indicate that an update is ready. Click the update button to pull the latest downloader files. If your software is current, the button changes to Launch, which opens your primary application. You can view your current build number in the settings menu at any time.

🛡️ Security

This loader only accesses the official repository to verify build numbers. It does not collect personal data from your machine. All file transfers occur through encrypted channels to ensure the integrity of your download. If your antivirus software flags the tool, you may need to add it to your list of exclusions to allow proper connection to the update server.

❓ Troubleshooting

If the hub fails to start, restart your computer and try again. Ensure your internet connection permits access to the GitHub servers. If the update fails, check your folder permissions to verify that you have authority to modify files in the installation directory. Most errors happen when the downloader folder is set to read-only mode.
